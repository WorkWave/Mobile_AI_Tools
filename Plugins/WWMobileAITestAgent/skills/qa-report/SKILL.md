---
name: qa-report
description: Generate a timestamped QA report markdown file after any test run (ticket test, POM run, workflow test, or regression). Includes session info, summary table, per-finding details, crash logs, and QA Lead recommendations.
---
> **Output directory:** Read `OUTPUT_DIR` from session memory (set by session-wizard Step 0). Use `$OUTPUT_DIR/<project>/` for all file paths instead of `projects/<project>/`.


# QA Report Generation

## Output Path

`$OUTPUT_DIR/<project>/reports/qa_report_<YYYY-MM-DD_HH-MM>.md`

Create the `reports/` folder if it doesn't exist.

## Report Template

```markdown
# QA Report — <Project> — <YYYY-MM-DD HH:MM>

## Session Info

| Field    | Value |
|----------|-------|
| Project  | <project> |
| Platform | <android \| ios> |
| Device   | <device name> (<udid>) |
| Build    | <version> (build <number>) |
| Source   | <Firebase \| Local> |
| Account  | <credential label> |
| Mode     | <Ticket-Driven \| POM \| Manual \| Regression> |
| Ticket   | <JIRA-ID> (if applicable) |
| Duration | <Xm Ys> |

---

## Summary

| Total | ✅ Passed | ❌ Failed | ⚠️ Blocked | 💥 Crashes |
|-------|-----------|-----------|------------|------------|
| N     | N         | N         | N          | N          |

**Overall verdict:** PASS / FAIL / PASS WITH ISSUES

---

## Findings

### ✅ <Test/AC Name>
> Steps performed: ...
> Expected: ...
> Actual: matches expectation ✅

---

### ❌ <Test/AC Name>
> Steps performed: ...
> Expected: ...
> Actual: ...
> Screenshot: `reports/screenshots/<name>.png`

---

### 💥 Crash — <Screen Name>
> During navigation to: <route>
> Device log excerpt:
> ```
> <last 20 lines of crash log>
> ```
> Screenshot: `reports/screenshots/crash_<timestamp>.png`

---

## Crash Log Summary

<If no crashes: "No crashes detected during this test run.">
<If crashes: paste full crash block with file path>

---

## QA Lead Recommendations

<For each FAIL and CRASH, provide:>
- **Root cause hypothesis**: ...
- **Suggested fix**: ...
- **Regression risk**: Low / Medium / High
- **Priority**: P1 / P2 / P3

---

## Test Artifacts

| Artifact | Path |
|----------|------|
| Navigation map | `$OUTPUT_DIR/<project>/nav_maps/navigation_map.json` |
| Screenshots | `$OUTPUT_DIR/<project>/reports/screenshots/` |
| Android logcat | `/tmp/regression_logcat.txt` (if regression) |
| iOS log | `/tmp/regression_ios.txt` (if regression) |

---

*Generated by WWMobileAITestAgent v1.0.0*
```

## After Writing Report

Display:
```
✅ QA Report generated:
   $OUTPUT_DIR/<project>/reports/qa_report_<timestamp>.md

Summary: <N> passed, <N> failed, <N> crashes
```

Ask: "Do you want me to create a Jira bug report for each failure? (yes/no)"
- If yes → for each FAIL/CRASH, use Atlassian MCP:
  ```
  mcp__claude_ai_Atlassian__createJiraIssue({
    project: "<project key>",
    summary: "Bug: <finding name>",
    description: "<steps + expected + actual + screenshot link>",
    issuetype: { name: "Bug" },
    priority: { name: "<P1|P2|P3>" }
  })
  ```
  Report the created issue keys.
